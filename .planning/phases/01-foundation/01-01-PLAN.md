## Trust Boundaries

| Boundary              | Description                                                             |
| --------------------- | ----------------------------------------------------------------------- |
| client→API            | Browser-to-API communication requiring authentication                   |
| API→database          | Supabase API calls requiring service role or authenticated user context |
| public→private routes | Middleware enforcement boundary                                         |

## STRIDE Threat Register

| Threat ID | Category | Component                | Disposition | Mitigation Plan                                                                |
| --------- | -------- | ------------------------ | ----------- | ------------------------------------------------------------------------------ |
| T-01-01   | S        | Authentication endpoints | mitigate    | Implement rate limiting on auth endpoints using Supabase built-in protections  |
| T-01-02   | T        | User session             | mitigate    | Use HTTP-only cookies for session storage, implement proper session validation |
| T-01-03   | I        | SQL injection            | mitigate    | Use Supabase parameterized queries, validate all inputs                        |
| T-01-04   | R        | Authentication bypass    | mitigate    | Middleware validates session on every protected route request                  |
| T-01-05   | D        | Account enumeration      | mitigate    | Use generic error messages, implement consistent timing for auth operations    |
| T-01-06   | E        | Client-side auth bypass  | mitigate    | All protected routes validated by middleware, not client-side checks only      |

</threat_model>
