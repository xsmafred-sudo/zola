| user→chat                     | User commands entered in chat interface                |
| chat→discovery agents         | Chat invoking discovery agents for data retrieval      |
| discovery agents→web          | Agents making HTTP requests to LinkedIn and Crunchbase |
| discovery agents→lead library | Agents storing discovered data in user library         |

## STRIDE Threat Register

| Threat ID | Category | Component                | Disposition | Mitigation Plan                                                                           |
| --------- | -------- | ------------------------ | ----------- | ----------------------------------------------------------------------------------------- |
| T-04-01   | S        | Command injection        | mitigate    | Validate and sanitize all discovery commands; use allowlist for permitted operations      |
| T-04-02   | T        | Data integrity           | mitigate    | Validate discovered data structure before storage; use checksums for critical fields      |
| T-04-03   | I        | Web scraping abuse       | mitigate    | Implement rate limiting per service; respect robots.txt; use official APIs when available |
| T-04-04   | R        | Unauthorized data access | mitigate    | Discovery agents only access public profile data; no private information extraction       |
| T-04-05   | D        | Resource exhaustion      | mitigate    | Rate limit discovery invocations per user; implement timeout controls for web requests    |
| T-04-06   | E        | Malicious data injection | mitigate    | Sanitize all scraped data before storage and display; prevent XSS in lead library views   |

</threat_model>
