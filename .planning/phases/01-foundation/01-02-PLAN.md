## Trust Boundaries

| Boundary     | Description                              |
| ------------ | ---------------------------------------- |
| user→browser | User-generated content displayed in chat |
| browser→API  | Chat messages sent to backend services   |
| API→database | Message storage and retrieval            |

## STRIDE Threat Register

| Threat ID | Category | Component                    | Disposition | Mitigation Plan                                                                                      |
| --------- | -------- | ---------------------------- | ----------- | ---------------------------------------------------------------------------------------------------- |
| T-02-01   | S        | Message input                | mitigate    | Sanitize all user input before display and storage; use React's automatic escaping                   |
| T-02-02   | T        | Message integrity            | mitigate    | Use message IDs and timestamps to detect tampering; validate message sequence                        |
| T-02-03   | I        | XSS via message content      | mitigate    | Sanitize all message content before rendering; use dangerouslySetInnerHTML only with trusted content |
| T-02-04   | R        | Unauthorized message sending | mitigate    | Middleware checks authentication before accepting message submissions                                |
| T-02-05   | D        | Message flooding             | mitigate    | Implement rate limiting on message submission per user/session                                       |
| T-02-06   | E        | Client-side message spoofing | mitigate    | All messages validated by backend; client cannot forge system messages                               |

</threat_model>
